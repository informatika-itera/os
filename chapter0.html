01

Basic Concept

Architecture vs Organization

Arsitektur vs Organisasi

Arsitektur

Apa yang terlihat oleh programmer (set instruksi, jumlah bit data, mekanisme I/O).

Ex: x86, ARM, RISC-V

Organisasi

Bagaimana fitur tersebut diimplementasikan secara hardware (sinyal kontrol, teknologi memori).

Ex: Cache L1 size, Clock speed

Konteks OS

Sistem Operasi harus paham keduanya untuk mengelola resource secara efisien dan menyediakan abstraksi yang tepat.

02

Von Neumann

The Stored Program Concept

Arsitektur Von Neumann

Konsep Kunci

Data dan Instruksi program disimpan di memori yang sama (Stored Program Concept).

Siklus Eksekusi (The Giant Loop)

CPU bekerja dalam loop yang tak henti:

1

Fetch

Ambil instruksi dari memori (berdasarkan Program Counter).

2

Decode

Terjemahkan instruksi. Apa yang harus dilakukan?

3

Execute

Jalankan! (Bisa baca/tulis memori, hitung matematika, lompat ke instruksi lain).

Hierarki Memori

Trade-off: Kecepatan vs Kapasitas vs Harga

Registers
L1/L2/L3 Cache
Main Memory (RAM)
Storage (SSD/HDD)
OS Role: Mengelola perpindahan data antar level ini (Caching, Virtual Memory) agar user merasa memori "cepat dan besar".

Mengapa OS Peduli?

  • Interrupts: CPU tidak bisa terus-terusan nge-cek I/O (polling). Device mengirim sinyal "Interrupt" ke CPU saat selesai. OS harus menangani ini.
  • Protection: User mode vs Kernel mode. OS menggunakan fitur hardware untuk mencegah program mengakses memori program lain.
  • DMA (Direct Memory Access): Hardware kusus untuk transfer data Disk <-> RAM tanpa membebani CPU. OS yang mengatur transfernya.
