Apa itu Sistem Operasi?

Definisi Sederhana

Sistem Operasi (OS) adalah Manajer Sumber Daya yang mengatur CPU, Memori, dan Disk.

OS menyediakan "tombol ajaib" (system calls) untuk aplikasi.

Tugas Utama:

  1. Memastikan sistem beroperasi benar.
  2. Menjalankan program dengan efisien.
  3. Abstraksi hardware yang rumit.

02

Virtualisasi

CPU Time Sharing & Memory Address Space

Virtualisasi CPU

Ilusi Banyak CPU

OS mengubah satu CPU fisik menjadi seolah-olah ada tak terbatas CPU virtual.

Teknik: Time Sharing

OS menjalankan proses bergantian dengan sangat cepat. Kita merasa semuanya jalan bersamaan (musik + ketik + download), padahal CPU "loncat-loncat".

Virtualisasi Memori

Ruang Pribadi

Setiap program merasa memiliki lokasi memori sendiri yang besar (Address Space), padahal fisiknya berbagi di RAM yang sama.

Masalah Konkurensi

Kerja Bareng Itu Susah!

Masalah rumit muncul saat mengerjakan banyak hal sekaligus pada data yang sama.

Analogi: Dua orang mencatat skor di papan tulis yang sama secara bersamaan.

// Thread 1
counter = counter + 1;
--- Context Switch ---
// Thread 2
counter = counter + 1;

Desain & Sejarah

Evolution of Operating Systems

Tujuan Desain OS

The Triad

Kinerja ‚Ä¢ Perlindungan ‚Ä¢ Keandalan

1. Abstraksi

Membuat hardware rumit jadi mudah dipakai.

2. Kinerja

Minimalkan overhead. OS jangan jadi beban.

3. Perlindungan

Isolasi program jahat. OS tidak boleh crash cuma karena 1 app error.

Evolusi Sistem Operasi

1950s Human Operator

Mainframe raksasa. Single user. Setup manual lama. CPU sering idle.

1960s Batch & Multiprogramming

Batch jobs. Mulai load banyak program ke memori agar CPU sibuk.

1970s Time Sharing & UNIX

Interaktif! User merasa punya mesin sendiri. Lahirnya UNIX.

Evolusi Sistem Operasi (Lanjutan)

1980s Personal Computer (PC)

Komputer murah. GUI muncul (Mac/Windows). Awalnya fitur OS mundur (demi performa), lalu maju lagi.

Now Mobile & Cloud

Android/iOS (Power efficiency). Cloud/Virtualization (OS di atas OS).
